## 상민

### Idea

- 유튜브 알고리즘을 분석해서 최신 트렌드 파악

### 생각해볼 점

- 데이터를 어떻게 구할지 (Hadoop을 사용해야될 만큼 빅데이터를 구할 수 있나?)
- 유튜브 알고리즘은 개인화되어서 단순히 소수의 사람들의 데이터로는 최신 트렌드를 파악하기 어렵다.
- 분석 방법, 실현 가능성
- 향후 분석 결과를 어디에 사용할지

## 박지수

### Idea

- 글을 보고 AI가 쓴 글인지 인간이 쓴 글인지 판별하는 AI 개발
- 데이터는 ChatGPT API, 구글/네이버 크롤링으로 직접 구하기

### 생각해볼 점

- 어떤 글 주제로 할지 (kaggle → essay 판별 | 데이콘 → 리뷰 판별)
- Hadoop을 사용해야될 만큼 빅데이터를 구할 수 있나? (데이터를 구하기까지 얼마나 걸릴지)
- 분석 방법, 실현 가능성
- 향후 분석 결과를 어디에 사용할지
    - 기업에서 데이터 플랫폼에서 이런 유형의 competition을 여는 것은 자연어 AI 모델에 활용하기 위함이다. (ex. GAN 모델 - 인간과 유사한 글을 생성하는 모델 개발에서 Error를 측정하기 위해)
        
        그렇다면 우리는 이 모델과 ChatGPT API를 결합해서 인간처럼 글을 생성해주는 서비스를 개발한다. (auto gpt 같은 느낌) 
        
        예)
        
        1. GPT로 글 생성
        2. 개발한 모델을 통해 각 문장별로 에러를 측정
        3. 측정한 결과를 바탕으로 GPT로 글 생성
        4. 2, 3번 반복
        5. 에러가 threshold 이하로 내려가면 중단 → 이렇게 해서 최대한 인간이 쓴 것 같은 글을 생성한다.
